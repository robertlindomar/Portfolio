Sobre mim

Trabalho com Java e Node.js no dia a dia e venho aprofundando Rust onde faz sentido para performance e segurança. Ao longo de vários sistemas reais, percebi um padrão incômodo: a mesma lógica de upload e armazenamento reaparecia projeto após projeto.

Em vez de repetir implementações, passei a tratar isso como problema de arquitetura: centralizar responsabilidades, padronizar contratos e permitir evolução sem acoplamento. Isso me levou a construir um microserviço em Rust para armazenamento de arquivos, com isolamento por projeto e autenticação por chave.

Hoje meu foco é backend, desenho de serviços e código reutilizável que outros times consigam consumir com clareza.

Formação: Análise e Desenvolvimento de Sistemas, UNIFUNEC — Santa Fé do Sul (conclusão prevista 2026). Atuação: desenvolvedor backend júnior na CodiTech desde outubro de 2024.

Projeto em destaque

Microserviço pensado para eliminar duplicação e padronizar armazenamento entre aplicações.

File Storage Service

Rust · API REST · armazenamento centralizado

Código no GitHub

Problema

Uploads e regras de armazenamento repetidos em várias aplicações, gerando manutenção dispersa, inconsistência e dificuldade para evoluir políticas (nomes, pastas, limites) de forma uniforme.

Solução

Microserviço dedicado ao armazenamento de arquivos, com API explícita para upload, download e metadados. Os demais sistemas passam a consumir um único serviço — um mini S3 interno, com contrato estável.

Tecnologias

Rust HTTP / REST Docker Persistência em disco (conforme desenho)

Diferenciais

  • Substitui uploads duplicados: uma implementação, vários clientes.
  • Multi-tenant por projeto ou cliente, com segregação de dados.
  • API key por projeto para autenticação e controle de acesso.
  • Base única para políticas de armazenamento, nomes e limites.
